C++入门学习笔记总结

2020-08-20
18k- 32 mins.
学习资料-C+++++++

###const:
const int p :常量指针，指针所指的值不可变，指针指向可变，即p = 20报错，p = &b 正确
短句方式：const int 的指针*p

int * const p :指针常量，指针指向不可变，指针所指值可变，即*p = 20 正确，p = &b 错误
断句方式 int * 的const p

const int * const p ：指针指向和所指值都不可变

总结断句方式，const + 后面一个单词与其他分开后中间加个“的”即可秒懂
记名方式：const = 常量，* = 指针，const int *p翻译为：常量指针，int * const p翻译为： 指针常量。

是否可调用判断：看const后面跟的先是还是p，例如：const int *p跟的是那么*p = 20；错误
而int * const p后面跟的是p，那么p = &b错误

将数组或结构体作为参数时用const修饰是为了节省空间和时间，穿地址是为了不将整个参数的内容复制一遍，加const是为了不让函数修改实参内容。Void Func（const struct student * stu）

###堆栈：
堆区是程序员负责开辟和释放的（自己new的东西）
栈区是编译器自动开辟和释放的
new运算符是在堆区创建一个空间并返回地址，需要用指针来接收，如：int* h; h = new int(10);

###函数重载：
可以通过参数类型和参数数量和参数顺序来区分，但是不可以通过返回值来区分，例如：
Func（int a）{return a;}和Func（int a）{double b; return b;}的入口都是Func(int),编译器不知道去哪个，有二义性。

###初始化参数：
参数可初始化如Func（int a= 10）,多个参数初始化的话，从第一个有初始化的参数后的其他参数都要有初始化,如Func（int a,int b = 10,int c =20）
而Func(int a,int b =10, int c)这样的则是错误的。

###占位参数：
例如：Func（int）,占位参数也可初始化如Func（int = 10）。

###作用域：
Public成员: 类内可以访问 类外也可以访问
Protect成员: 类内可以访问 类外不可以访问
Private成员: 类内可以访问 类外不可以访问
即在主函数中创建一个对象，也无法通过对象来对protect成员和private成员进行操作。
成员属性私有化：可以把类内的变量都放到private里，再在public里设置get()和set()方法来控制读写权限。

###struct和class
struct和 class的唯一区别是默认权限不同，struct默认是public，class默认是private
即class a{int a;//默认包含在private里}; 而struct{int a;//默认是public成员};

###默认包含的函数：
构造函数：分为普通构造函数和拷贝构造函数，无返回值也没有void ，普通构造有参无参均可，没有写的话系统默认添加空的构造函数和析构函数。

析构函数：释放自己写的堆区的数据，在对象销毁时执行，在main函数中的对象要到main函数执行完后执行相应的析构函数。通常在里面释放自己new的指针并且为了防止野指针出现，进行置空操作
如：~Person(){if(p != NULL){delete h;} p = NULL;}

拷贝构造函数：Student(const Student &s){age = s.age…},加const是为了不把本体改了，用引用是为了节省空间时间。

构造函数的调用:1.括号法 调用方法为 Person p(10);或Person p(p1) 2.显示法：Person p = Person(10);或Person p = Person(p1); 3.隐式转换法：Person p = 10;或Person p = p1;

注意如果要调用无参构造方法时不能加()，如:Person p(); 否则编译器会认为是一个函数的声明，即返回类型为Person的p()方法。
而显示法的右值Person(10)其实是匿名类，而显示法可以看成是是将这个匿名类命名为p，如果匿名对象单独一行 如：Person(10);的话，当前行执行后会立即回收掉匿名对象，这也是匿名类的特点之一。(PS：不回收也用不了啊，没有名怎么调用)（回收速度之快甚至在下一行执行前就回收了）
不要用拷贝构造函数来初始化匿名对象 如：Person(p); 因为编译器会等价地认为Person(p) 就是Person p。即重新定义一个所拷贝的对象p，会有重定义报错。
隐式转换法编译器会将Person p = 10; 隐式转换成 Person p = Person(10); (编译器贴心地偷偷给你加小括号i了i了)

用值传递的方式给函数参数传对象时，调用拷贝函数复制一份对象传递过去。
如void work(Person p){} 若其他函数调用了work函数 如：work(p);则实际上传过去的是通过拷贝构造函数拷贝的副本而不是原来的对象p。

若局部对象作为返回值时 如Person work(){Person p; return p;}也调用拷贝构造函数复制一份局部对象的副本用于返回，因为局部对象在该函数执行完成后会释放掉，而将拷贝出来的对象才是返回去的那个，有些编译器会进行优化，直接将返回值赋给接收的对象而省略拷贝函数这一步。

当自己编写了默认构造函数时，编译器会提供拷贝构造函数和析构函数，而当自己写了拷贝构造函数时则只提供析构函数，不提供默认构造函数，如编写了Person(const Person &p){…} 而不写默认构造函数，则调用Person p;//调用默认构造 则会报错。
总结就是：
用户定义有参 则不提供默认无参 但提供默认拷贝构造和析构
用户定义拷贝 则不提供默认无参和拷贝 只提供析构

###深拷贝与浅拷贝
浅拷贝与深拷贝：当利用浅拷贝去复制数值时，没有问题，但当用浅拷贝字符串常量和堆区指针时，则会在析构时出现重复释放报错的问题。深拷贝就是在堆区重新开辟空间放这些东西而不会重复释放。如：Person(const Person &p){age = p.age; h = new int(p.h);}
总结：*如果属性有在堆区开辟的则要自己提供拷贝构造函数。**

###初始化列表：
Person{public: int X;int Y;int Z;}
Person（int a, int b, int c）:X(a),Y(b),Z(c){…}X初始化为a以此类推

###类对象作为类成员：
例如B类里面可以有一个A类的a的对象Class B{A a;} 而且当实现B类对象时会先创建A类对象而不是先创建B类对象。并且析构时先析构B类对象，再析构A类对象
即：A构造，B构造，B析构，A析构的顺序（有点入栈出栈的味道）

###静态成员变量和静态成员函数：
访问静态成员函数有两种方式，实例化一个对象来访问Person p; p.func(); 或者通过函数名来访问 Person::func();
而静态成员函数里只能访问静态成员变量，如果里面访问的是非静态的成员变量（即每个实例化都有的对象，p1有age，p2也有age）则编译器不知道你访问的是哪个实例化对象的非静态成员变量而报错。静态成员函数也有访问权限，如写在private:下面，则类外也无法通过函数名等方式来访问。

静态成员变量：无论作用域是public还是private都必须在类内声明，在类外初始化，否则无法访问该内存/变量。
初始化方式：int Person::number = 100;//记得要加类名

空对象：空对象会占用1个字节的大小，用来区分多个不同的空对象，区分空对象占内存的位置

成员变量与成员函数储存关系：成员变量与成员函数是分开存储的，在类内无论写多几个静态还是非静态的func()方法 都不会影响sizeof(p)所输出的大小。只有声明非静态的成员变量才会改变，即静态成员变量和所有成员函数是不属于类的实例化对象上的。(PS：对象有的只是非静态成员变量，有点孤独寂寞冷)

###链式编程思想：
当调用p2.PersonAddAge(p1).PersonAddAge(p1).PersonAddAge(p1)时，PersonAddAge()方法应该返回引用而不是返回对象，如果返回对象的话会调用拷贝构造函数生成新对象用于返回，所以每次改变的age都是新对象的age，导致最后的结果相当于只执行了一次PersonAddAge(p1)。（PS：返回当前调用对象地址或引用用return *this;返回）

###常函数与常对象：
常函数在函数后面加const 如：void work() const ，常对象在前面加const 如：const int a; 常函数内不可修改成员变量（用mutable 修饰的变量除外），常对象只能调用常函数不能调用普通函数，普通对象可以调用两种函数。

###全局函数作友元：
在某类里声明该函数并加前缀friend即可
如：class A{friend void work(Person *p);};
类作友元：在某类里声明该函数并加前缀friend即可
如：class A{friend class B;};
成员函数作友元：在某类里声明该函数并加前缀friend即可
如：class A{friend void GoodGay::visit();};

##运算符重载：
成员函数重载运算符和全局函数重载运算符
全局函数重载形式：Person operator+ (重载参数，重载参数)
成员函数重载形式：Person operator+ (重载参数）
内置数据类型int double等不能重载
遇见的坑：
1.指针作参数时传地址，引用作参数时直接传对象
2.不能用两个指针作为重载参数，不符合重载规则，不过可以一个是对象，一个是指针，也可以是一个是引用，一个是指针。（PS:两个指针相加好像也是内置的的数据类型，所以不能重载）

###重载<<运算符：
只能用全局函数重载形式，如果用成员函数重载，如void operator<<(cout)则只能用p << cout;的形式来输出，而不是cout << p; 全局重载形式如下：
ostream& operator << (ostream & cout, Person & p) {
cout << p.a;
return cout;
}
其中cout是全局变量所以用cout的引用&cout，而cout是ostream类，为了使普通输出，
如：cout << “a” << p << “s” << endl;的链式编程不断掉，只能同样返回cout 使cout << “a”之后返回一个cout 拼接成 cout << p << “s” << endl;的形式。（PS：参数里面的cout可以随便改成别的，因为引用就是起别名，不过调用的时候记得用别名就是了）

###重载前置++运算符：
MyInteger& operator++(){…;return *this;}返回引用是为了只对同一个数进行操作，以防返回对象时用拷贝构造函数导致返回的不是原来的，导致++(++(++x))之后返回的是3，而再次读取的时候发现原来的数据仍然只是1，只有第一次++是加到原数据上的，后两次++都是加到拷贝的新的对象上。

###重载后置++运算符：
格式与前置稍有不同 如：myint operator++(int); int是占位参数，用于区分前置和后置重载，写double都不好使，只能写int。先记录结果，再递增数据，最后返回先前的结果实现后置++的效果。如：myint temp = *this; this->a++; return temp;(注意：后置++要返回对象而不是返回引用，如果返回引用则返回的是局部的引用)
总结：前置递增返回引用，后置递增返回值

###赋值运算符（=）重载：
person& operator=(Person &p){
//先判断是否有属性再堆区，如果有先释放干净,此处age是指针指向堆区数据
if(age != NULL){delete age; age = NULL;}
//然后进行深拷贝
age = new int(*p.age);
//最后要返回当前类型的引用，因为原始赋值运算符允许 a = b = c的链式编程
Return *this;
};

###重载函数调用运算符，即小括号，也称为仿函数：
void operator()(参数列表){…}，仿函数非常灵活，里面想实现什么都行，加法减法什么的都行，调用起来也很方便如：
MyAdd myadd;
myadd(10,20);//看上去特别像函数调用所以成为仿函数,实际上是实例对象后面跟了个括号
MyAdd()(100,100);//也能调用该仿函数，原理是前面的MyAdd()生成了一个匿名对象。

###不同的继承方式：
公有继承：父类的private不可访问，其他权限不变的继承下来
保护继承：父类的private不可访问，父类的public和protect继承到子类时变成protect
私有继承：父类的private不可访问，父类的public和protect均继承到子类的private里去

###继承中的对象模型：
父类中所有非静态的成员属性都会被子类继承下去，甚至是父类的private，其实父类private里的成员属性是被编译器隐藏了，所以是“访问不到”而不是不存在。

###继承中构造和析构顺序：
父类构造–子类构造–子类析构–父类析构

###继承同名成员处理方式：
子类直接访问，父类加作用域，无论是函数还是变量，而且当子类中有父类同名成员函数时，父类中所有同名函数（包括重载的）都会被编译器隐藏，想调用父类同名函数只能通过加作用域的方式调用。

###菱形继承：
当菱形继承导致有两份相同数据时，利用虚继承来解决问题Calss A:virtual public B{};

###虚继承：
当某类虚继承时他从基类里继承的是虚基类指针(vbptr)，指针指向虚基类表(vbtable),虚基类表中记录的是偏移量，当该函数从两个基类继承了两个相同属性时，则两个指针位置+偏移量 = 同一个属性

###多态：
静态多态即函数重载和运算符重载，动态多态
子类可以自动转父类，即可以用父类指针接子类对象，Animal &animal = cat;
而此时执行doSpeak(cat);执行的却是父类的：doSpeak(animal),因为地址早绑定 在编译阶段就确定了函数地址
解决方法：在doSpeak前加virtual，如 virtual void doSpeak(Animal &animal){} 虚函数是地址晚绑定。
动态多态满足条件：1、有继承关系 2、子类要重写父类的基函数
多态使用条件：父类指针或引用指向子类对象，如：AbstractCalculator *cla = new AddCalculator;

###纯虚函数：
也就是抽象类，如：virtual void work(int a) = 0; 无法实例化抽象类的对象，子类必须重写纯虚函数

###虚析构和纯虚析构：
父类指针（指向子类）在析构的时候不会调用子类中的析构函数，导致子类如果有堆区的属性，会出现内存泄漏。
解决方法：将父类的析构改为虚析构/纯虚析构
若一个类有纯虚析构，则该类也算抽象类，纯虚析构与虚析构都需要具体实现。
总结：虚析构和纯虚析构就是用来解决通过父类指针释放子类对象不干净的问题，子类中没堆区数据则可以不写虚析构或纯虚析构，拥有纯虚析构的类也是抽象类

###文件操作：
进行文件操作要包含头文件,操作文件三大类：ofstream写操作，ifstream读操作，fstream读写操作

类模板分文件编写以及解决方法：
1.include xxx.cpp文件，即类模板函数实现的那个文件而不是头文件，因为类模板的成员函数是调用时生成的，而不是编译时生成的，所有如果包含的是头文件，则编译器在链接阶段（此时还没调用方法，所以不会有成员函数的具体实现）看不到cpp中的具体实现而报错。

##面向对象编程：
面向对象编程的三大特点：封装、继承、多态
STL广义上分为：容器、算法、迭代器
STL六大组件：容器、算法、迭代器、仿函数、适配器、空间配置器
1.容器：各种数据结构，如vector、list、deque、set、map等，用来存放数据
2.算法：各种常用的算法，如sort、find、copy、for_each等
3.迭代器：扮演容器与算法之间的胶合剂
4.仿函数：行为类似函数，可作为算法的某种策略
5.适配器：一种用来修饰容器或仿函数或迭代器接口的东西
6.空间适配器：负责空间的配置与管理

序列式容器：强调值的排序，容器中每个元素均有固定位置（放进去是什么顺序，存储就是什么顺序）
关联式容器：二叉树结构，各元素之间没有严格的 物理上的顺序关系

质变算法：运算过程中更改区间内元素的内容，如替换、拷贝、删除等
非质变算法：和上面相反，如查找、计数、遍历、寻找极值等

###容器：vector
算法：for_each
迭代器：vector::iterator
第一种遍历方法：
vector v;
vector::iterator itBegin = v.begin();//起始迭代器，指向容器中第一个元素
vector::iterator itEnd = v.end();//结束迭代器，指向容器中最后一个元素的下一个位置

扩展空间时并不是在原有的空间内进行拓展，而是直接分配更大的空间将原有的数据转移后释放掉原有数据

cout << v1.capacity() << endl;//输出容量
cout << v1.size() << endl;//输出实际大小（注意capacity >= size）
v1.resize(15);//重新定义大小，多出的位置默认填充0

###set/multiset容器：

set不允许重复元素，而multiset允许重复元素，两个都包含在set头文件中（set即使插了也不会有重复的元素，可以插，但插不进，听起来怪怪的），其底层是二叉树实现，所以插进去之后就自动被排序
set没有resize(),因为resize()会有重复元素//空位置默认零的规则，set在插入时会返回一个pair的对组（包含一个迭代器和一个bool类型），其中bool用于区别是否插入成功

而multiset不会有这个插入重复检测，只要是符合的数据类型都能插
